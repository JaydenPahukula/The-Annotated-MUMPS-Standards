3.3.13 Key and Pointer Button Codes

Key sequences, consisting of one or more keys, are identified by key codes (keycodes).

    keycode::= noncommachars

Pointer buttons are identified by pointer button codes (pbuttoncodes).

    pbuttoncode::= noncommachars

The correspondence between actual keyboard keys and key codes and between actual pointer buttons and pointer button codes is established by a mapping mechanism. For each key code, the mapping mechanism specifies the corresponding key sequence and whether the key code is a data key code or a function key code. For each pointer button code, the mapping mechanism specifies the corresponding pointer button.

While this International Standard does not specify how this mapping is carried out, a mapping mechanism must be available.
