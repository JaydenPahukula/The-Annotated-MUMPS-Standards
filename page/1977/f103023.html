3.3.20 Data Exchange

deaction identifies an action to perform over a Data Exchange link

    deaction::= expr V │
│
│
│		 SEND
REQUEST
EXECUTE		 │
│
│
│

destatus identifies the outcome of an action performed over a Data Exchange link

    destatus::= expr V │
│
│
│		 FAILED
SUCCESS
TIMEOUT		 │
│
│
│

delinkid is a platform-specific identifier for a data exchange link. A delinkid must be a valid M[UMPS] subscript, and each link in which an M[UMPS] process is concurrently involved must yield a unique identifier.

    delink::= chars

deitem identifies an item within a data source

    deitem::= chars

demode defines the operating mode of a Data Exchange link

    demode::= expr V │
│
│
│
│		 AUTOMATIC
MANUAL
NONE
NOTIFY		 │
│
│
│
│

derole defines the role of the M[UMPS] process in a Data Exchange link

    derole::= expr V │
│
│		 CLIENT
SERVER		 │
│
│

deservice identifies the service name under which a display acts as a data exchange source.

    deservice::= chars

desource identifies a data exchange source available to the M[UMPS] process. Each such desource must be a valid M[UMPS] subscript.

    desource::= chars

detopic identifies the topic name under which a window acts as a data exchange source.

    detopic::= chars
