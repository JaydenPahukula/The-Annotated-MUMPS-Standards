B.8.1 Device parameters

M[UMPS] processes set device parameters with the OPEN, USE, and CLOSE commands. If the meaning of a parameter differs when used with different commands, the server’s implementer should define different device parameters for the different uses.

The M[UMPS] language defines 2 permissible styles of deviceparam, and the server defines which 1 applies to a device. The M[UMPS] syntax for deviceparam in the OPEN command can be ambiguous if the client’s parser does not know which style applies, so the designers of OMI anticipate the following sequence of operations in the client:

  1. Parse the device name and mnemonicspec of the OPEN command.
  2. (required only if the style is unknown) Request get device parameters to find which style the server requires.
  3. Parse the deviceparams.
  4. Request open device to the server.
