2.2 External routines and names

The externalroutinename namespace is unspecified, as this is a function of the binding, although at the present time, a maximum of twenty-four (24) characters allowed is placed upon externalroutinenames to be treated uniquely, although this should be viewed as a minimum number that needs to be handled rather than as the maximum number that can be used. Any number of characters, from one to the maximum number shall be valid as externalroutinenames. Any additional external mapping between these names and any actually used by an external package is an implementation issue.
