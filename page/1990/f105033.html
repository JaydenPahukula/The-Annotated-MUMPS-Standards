5.4.27 PSELECT wattribute

Definition

For a build-mode window PSELECT specifies the gadget which is the principal selection. When more than one gadget is selected they may be moved as a group but only one, the principaly selection, may be resized. If PSELECT is undefined, no gadget is selected.

If an application assigns a value to PSELECT which identifies a gadget on the window, that gadget becomes the principal selection and is added as a descendant to the VALUE wattribute if necessary. If an application assigns a value to PSELECT which does not identify a gadget which is a child of the window, an error condition occurs with $ECODE containing ",M48," and $EREF containing a value that indicates the ssvn reference for which the error occurred.

If an application kills the PSELECT wattribute it becomes undefined and the VALUE wattribute is also implicitly killed.

For a window that is not in build mode, the wattribute may be set and referenced but is ignored.
