B.8.3 Delayed write completion

Many devices take substantial time to return the completion status of a write operation, or even to indicate their current status. Therefore, to avoid blocking the protocol, 5.4.28 requires an immediate response by the OMI server showing that the status is not yet available. The agent should not request status routinely, but only when the client process needs it, and should repeat the request at suitable intervals until the server returns the status.
